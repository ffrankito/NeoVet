## System Overview

### Description

A stateless web chatbot that answers FAQ questions for NeoVet clinic clients. The user sends a message via a web widget; the system streams a response generated by Claude claude-sonnet-4-6 using a system prompt seeded with real clinic data. No data is persisted. No external systems are called.

### Architecture Diagram

```mermaid
graph TD
    User["Client (browser)"] -->|chat message| Widget["Web Chat Widget\n(Next.js)"]
    Widget -->|POST /api/chat| API["API Route\n(Next.js App Router)"]
    API -->|streamText()| Claude["Claude claude-sonnet-4-6\n(Anthropic API)"]
    Claude -->|streamed response| API
    API -->|stream| Widget
    Widget -->|rendered reply| User
```

## AI / LLM Usage

### Chat response

| Field | Value |
|---|---|
| **Model** | `claude-sonnet-4-6` |
| **Calling pattern** | `streamText()` — streaming |
| **Prompt location** | `src/app/api/chat/route.ts` (system prompt inline or `src/lib/prompts/system.ts`) |
| **Output validation** | None — free-form text response |
| **Estimated tokens per call** | Input: ~800 (system prompt + history) / Output: ~200 |
| **Fallback on LLM error** | Surface user-friendly message in Spanish |

**System prompt strategy:** The system prompt is a static string seeded with Paula's real clinic data (hours, services, location, prices). It instructs the model to answer only questions about the clinic and to respond in Argentine Spanish. It does not use dynamic DB injection in v1.
