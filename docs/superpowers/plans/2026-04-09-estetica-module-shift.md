# Estética Module Shift — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform the grooming ("peluquería") module into a broader aesthetics ("estética") module with configurable service types, replacing the 3-tier pricing model with service-based pricing.

**Architecture:** Add a `serviceId` FK to `grooming_sessions` linking to the existing `services` table. Rename the `service_category` enum value `"peluqueria"` → `"estetica"` in Postgres. Remove `priceTier` from the session creation flow (column stays nullable for historical data). Update all user-facing text from "Peluquería" → "Estética" and "Peluquero/a" → "Esteticista". Seed 7 initial estética service types.

**Tech Stack:** Next.js 16 App Router, Drizzle ORM, Supabase (PostgreSQL), Tailwind CSS, shadcn/ui

Open the generated SQL file. Drizzle may try to drop and recreate the enum, which is destructive. Replace the enum-related statements with a safe rename. The final migration SQL should contain (in this order):

```sql
-- 1. Rename the enum value "peluqueria" → "estetica"
ALTER TYPE "public"."service_category" RENAME VALUE 'peluqueria' TO 'estetica';

-- 2. Add serviceId column to grooming_sessions

  ('svc_estetica_cortesan', 'Corte sanitario', 'estetica', 45, NULL, NULL, true, NOW(), NOW()),
```

## Task 11: GVet estética import script

**Files:**
- Create: `crm/scripts/import-estetica.ts`

- [ ] **Step 1: Create the import script**

```typescript
/**
 * Import estética (grooming) sessions from GVet CSV export.
 *
 * Expected CSV columns: Fecha, Cliente, Paciente, Tipo, Usuario, ¿Está pagado?, Costo
 *
 * Usage:
 *   npx tsx crm/scripts/import-estetica.ts path/to/estetica-export.csv
 */

import { parse } from "csv-parse/sync";
import { readFileSync } from "fs";
import { db } from "../src/db";
import { groomingSessions, groomingProfiles, services, patients, clients } from "../src/db/schema";
import { groomingSessionId, groomingProfileId } from "../src/lib/ids";
import { eq, and, ilike } from "drizzle-orm";

const csvPath = process.argv[2];
if (!csvPath) {
  console.error("Usage: npx tsx crm/scripts/import-estetica.ts <csv-path>");
  process.exit(1);
}

const raw = readFileSync(csvPath, "utf-8");
const records: Record<string, string>[] = parse(raw, {
  columns: true,
  skip_empty_lines: true,
  trim: true,
  bom: true,
});

async function main() {
  console.log(`Importing ${records.length} estética records...`);

  // Pre-load service map (name → id)
  const allServices = await db
    .select({ id: services.id, name: services.name })
    .from(services)
    .where(eq(services.category, "estetica"));
  const serviceMap = new Map(allServices.map((s) => [s.name.toLowerCase(), s.id]));

  // Pre-load patients with client names for matching
  const allPatients = await db
    .select({
      patientId: patients.id,
      patientName: patients.name,
      clientId: patients.clientId,
      clientName: clients.name,
    })
    .from(patients)
    .leftJoin(clients, eq(patients.clientId, clients.id));

  let imported = 0;
  let skipped = 0;

  for (const row of records) {
    const fecha = row["Fecha"]?.trim();
    const clienteName = row["Cliente"]?.trim();
    const pacienteName = row["Paciente"]?.trim();
    const tipo = row["Tipo"]?.trim();
    const costoRaw = row["Costo"]?.trim()?.replace(/[$.]/g, "")?.replace(",", ".");
    const costo = costoRaw ? parseFloat(costoRaw) : null;

    if (!fecha || !pacienteName) {
      console.log(`  SKIP (missing data): ${JSON.stringify(row)}`);
      skipped++;
      continue;
    }

    // Match patient by name + client name
    const match = allPatients.find(
      (p) =>
        p.patientName?.toLowerCase() === pacienteName.toLowerCase() &&
        (!clienteName || p.clientName?.toLowerCase() === clienteName.toLowerCase())
    );

    if (!match) {
      console.log(`  SKIP (patient not found): "${pacienteName}" / "${clienteName}"`);
      skipped++;
      continue;
    }

    // Match service type
    const svcId = serviceMap.get(tipo?.toLowerCase() ?? "");
    if (!svcId) {
      console.log(`  WARN (service type not found): "${tipo}" — importing without serviceId`);
    }

    // Parse date
    const parsedDate = new Date(fecha);
    if (isNaN(parsedDate.getTime())) {
      console.log(`  SKIP (invalid date): "${fecha}"`);
      skipped++;
      continue;
    }

    // Ensure grooming profile exists
    const [existingProfile] = await db
      .select({ id: groomingProfiles.id })
      .from(groomingProfiles)
      .where(eq(groomingProfiles.patientId, match.patientId))
      .limit(1);

    if (!existingProfile) {
      await db.insert(groomingProfiles).values({
        id: groomingProfileId(),
        patientId: match.patientId,
      });
    }

    // Insert session (no groomedById — we're ignoring GVet "Usuario")
    // Use a placeholder staff ID or the first available one
    await db.insert(groomingSessions).values({
      id: groomingSessionId(),
      patientId: match.patientId,
      appointmentId: null,
      serviceId: svcId ?? null,
      groomedById: "IMPORT_PLACEHOLDER",
      priceTier: null,
      finalPrice: costo ? String(costo) : null,
      beforePhotoPath: null,
      afterPhotoPath: null,
      findings: [],
      notes: `Importado de GVet. Tipo original: "${tipo}"`,
      createdById: null,
      createdAt: parsedDate,
      updatedAt: parsedDate,
    });

    imported++;
    console.log(`  OK: ${pacienteName} — ${tipo} — $${costo ?? "?"}`);
  }

  console.log(`\nDone. Imported: ${imported}, Skipped: ${skipped}`);
  process.exit(0);
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

**Important:** The `groomedById` is set to `"IMPORT_PLACEHOLDER"` because we agreed to ignore GVet's "Usuario" field. This column has a NOT NULL constraint, so we need a placeholder value. Before running this script, either:
- Create a placeholder staff entry for imports, OR
- Temporarily make `groomedById` nullable in the import

The implementer should verify which approach is best for the existing data. If there's already a generic admin/import staff ID, use that.
